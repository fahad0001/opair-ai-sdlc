# Architecture Decision Records (ADR)

This folder contains decision records for changes that affect architecture, dependencies, public APIs, security, or performance.

## When an ADR is required

Create an ADR when:

- Adding/removing a major dependency
- Changing folder boundaries or architectural rules
- Changing public APIs (frontend routes, backend endpoints, shared types)
- Introducing new persistence/storage layers
- Changing auth/authz patterns
- Accepting a tradeoff that impacts performance, security, cost, or maintainability

## Naming convention

- `ADR-0001-<short-title>.md`
- Increment numbers sequentially.

## ADR Template

- Use `ADR-template.md` for new decisions.
